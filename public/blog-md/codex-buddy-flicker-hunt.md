# ล่าต้นเหตุจอกระพริบ

> ปัญหาที่ดูเล็ก แต่ไล่อยู่ทั้งวัน — เพราะเราเดาจากอาการ ไม่ได้ดูของจริง

## เริ่มจากอะไร

เราอยากได้ **codex buddy** (ผู้ช่วย AI เขียนโค้ด) มานั่งข้าง Claude Code ทางขวา
พอเปิดขึ้นมา จอด้านขวากลับ **กระพริบปิ๊บ ๆ** ตลอดเวลา — น่ารำคาญมาก

## 5 ทฤษฎีที่ผิด

ไล่ทีละอัน ทุกอันฟังดูมีเหตุผล แต่ทดสอบแล้วผิดหมด:

| ทฤษฎี | ทำไมคิด | ผลจริง |
|---|---|---|
| `--madmax` สร้าง HUD | omx มี HUD feature | ❌ คนอื่นก็ใช้ ไม่กระพริบ |
| วิธี spawn (team up) | layout ต่างกัน | ❌ เขาก็ spawn มือเหมือนเรา |
| window height ≥ 45 | source บอกงั้น | ❌ จอเล็กกว่า 45 ก็ยังมี |
| pane height < 45 | ทดสอบดู | ❌ เตี้ยแล้วก็ยังมี |
| env `OMX_TMUX_HUD_OWNER` | gate ใน source | ❌ คนอื่นก็ตั้ง =1 |

## ตัวจริง

คนที่หาเจอจริง ๆ คือ **มนุษย์** — ชี้ให้ดูบรรทัดล่างสุด:

```
0 "⠐ Claude Code"
```

ตัว `⠐` คือ **spinner ของ Claude Code** (ตัวหมุนตอนคิด) ที่ tmux ดึงมาโชว์ที่
**เส้นขอบล่างของ pane** ผ่าน `pane-border-status bottom`

spinner หมุนทุก frame → tmux ต้องวาดเส้นขอบใหม่ทุก frame → **กระพริบ**

## แล้วมันมาจากไหน

ไล่ chain ต่อ พบว่า `maw tile` (คำสั่งเปิด pane) เป็นคนตั้ง `pane-border-status bottom`
ทุกครั้งที่รัน — เปิด border เพื่อ label ชื่อ pane แต่ผลข้างเคียงคือดึง spinner มาโชว์ที่ขอบ

## แก้

บรรทัดเดียว:

```bash
tmux set-option pane-border-status off
```

## บทเรียน

ที่ไล่ผิด 5 รอบ เพราะเรา **เดาจากอาการ** ("มันกระพริบ = คงเป็น HUD") แทนที่จะ
**ดูของจริง** (บรรทัดที่ตาเห็นจริง ๆ ว่าเขียนอะไร) — พอมนุษย์ชี้ให้ดูตัวหนังสือจริง
ก็เจอใน 5 นาที

> ดูของจริงก่อนเดา — telescope ที่ดีต้องเล็งให้ตรงก่อนจะสรุปว่าเห็นอะไร
